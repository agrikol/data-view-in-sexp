# Представление данных в S-выражениях на Python
## Требования
1. Определите формат представления данных с древовидной структурой в виде S-выражений. В качестве семантической основы рекомендуется рассмотреть форматы XML и/или JSON. Допускается использование как классических S-выражений, так и расширенных, используемых в языке Clojure.
2. Разработайте язык навигации и простых запросов для заданного формата. Должны поддерживаться: относительный или абсолютный путь к узлу дерева, путь с
условиями на свойства промежуточных узлов, путь с переменной вложенностью. В качестве семантической основы рекомендуется рассмотреть язык XPath для XML.
Реализуйте функции поиска и модификации заданного формата посредством этого языка.
3. Разработайте представление схемы (по аналогии с XML Schema). Реализуйте функции проверки документа по схеме.

## Дополнительные требования
4. Разработайте язык трансформации в HTML, как упрощенный аналог XSLT.
5. Визуализация дерева S-выражений.

## Реализовно
На данный момент реализация проекта включает пункты 1, 2, 3, 5 требований:
1. **Требование 1:**
- Документы [format.md](docs/format.md), описывающий основные правила формата; [spath.md](docs/spath.md), описывающий язык навигации SPath; [schema.md](docs/schema.md) описывающий представление схемы, а также [API.md](docs/API.md) с описанием API;
- Модель представления данных: Класс `Scalar` для представления скалярных величин и `Node` для представления AST-дерева;
-  Лексер и Парсер для корректного разбора AST на составляющие (классы `Lexer` и `Parser`);
2. **Требование 2:**
- Модель запроса `SPath`, реализация Лексера, Парсера, Движка: классы `SPathLexer`, `SPathParser`, `SPathEngine`;
3. **Требование 3:**
- Представление схемы, включая функции проверки документа по схеме: Класс `SchemaNode` для представления схемы, `Interpreter` для разбора и валидации схемы и класс `Validator` для проверки документа по схеме;
4. **Требование 4:** TBD
5. **Требование 5:**
- CLI-визуализация документа в виде разноцветного дерева (класс `TreeRenderer`), которая работает как с S-выражениями как в виде сырой Python-строки, так и с `Node`. Включает в себя работу как с Unicode, так и с ASCII форматами.;
6. Дополнительно: 
- Предоставление пользователю API, включающее методы: `loads`, `dumps`, `validate`, `tree`, `path`. Все методы содержат подробный `docstring`;
- Покрытие тестами всех основных классов и методов;
- Документы включают описание синтаксиса, грамматики, типы, примеры использования.
