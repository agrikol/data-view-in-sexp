# Формат представления данных в S-выражениях

## Основные правила формата
- Данные представляются как вложенные списки (Lisp-подобные S-выражения).
- Корневой элемент — список, где первый элемент — имя узла (строка), за ним следуют атрибуты (как ключ-значение пары) и дочерние узлы.
- Атрибуты: Представлены как ключ-значение пары в виде `(:key "value")` (используем двоеточие для ключей, как в Clojure).
- Дочерние узлы: Вложенные списки аналогичного формата.
- Узел может содержать скалярное значение, если он является листом (далее leaf values).
- Поддержка массивов: Если узел содержит несколько одинаковых дочерних, они просто повторяются в списке.

## Leaf values
### Допустимые типы:
- string
- number
- boolean
- null
### Ограничения
Узел либо:
- содержит скалярное значение
- содержит дочерние узлы
но не одновременно

## Пример эквивалента JSON в S-выражении
- JSON: {"person": 
            {"name": "Igor", 
            "age": 22, 
            "children": ["Ivan", "Petr"]
            }
        }
- S-выражение: (person 
                    (:name "Igor") 
                    (:age 22) 
                    (child "Ivan") (child "Petr"))

## Парсинг и представление в Python
- В Python данные хранятся как вложенные списки/кортежи.
- Рекомендуемый тип: list, где первый элемент — str (имя узла), затем чередование атрибутов (tuple как (:key, value)) и дочерних списков.
- Пример: data = ['person', (':name', 'Igor'), (':age', 22), ['child', 'Ivan'], ['child', 'Petr']]
